## 🎯 项目目标

在不改变现有 Flask 架构的前提下，通过增量开发方式，将系统从"旅游问答工具"升级为"旅游规划助手"。

### 开发原则

| 原则 | 说明 |
|------|------|
| ❌ 不重构 | 保持现有代码结构 |
| ❌ 不推翻现有代码 | 增量开发，逐步迭代 |
| ❌ 不引入复杂中间件 | 保持轻量级架构 |
| ❌ 不引入微服务 | 单体应用优先 |
| ✅ 优先提升用户体验 | 以用户价值为核心 |

## 📍 阶段一：V6.1 用户体验优化

**预计周期：** 3天  
**目标：** 让用户能够查看景点详情、收藏景点、查看浏览记录。

### 模块清单

| 模块 | 名称 | 核心功能 |
|------|------|----------|
| 模块1 | 景点详情页 | 展示景点完整信息（图片、简介、门票、开放时间等） |
| 模块2 | 收藏功能 | LocalStorage 实现景点收藏 |
| 模块3 | 最近浏览 | 记录用户浏览历史（最近20条） |

---

## 📍 阶段二：V6.2 AI 能力增强

**预计周期：** 4天  
**目标：** 让 AI 不仅回答问题，还能提供旅游建议。

### 模块清单

| 模块 | 名称 | 核心功能 |
|------|------|----------|
| 模块4 | AI 回答增强 | 统一 Prompt 模板，结构化输出 |
| 模块5 | AI 一日游 | 根据用户选择生成一日游行程 |
| 模块6 | 首页推荐卡 | 今日/周末/雨天推荐，路线推荐 |

---

## 📍 阶段三：V7 旅游规划能力

**预计周期：** 5天  
**目标：** 让系统具备简单旅游规划能力。

### 模块清单

| 模块 | 名称 | 核心功能 |
|------|------|----------|
| 模块7 | 景点图片库 | 每个景点至少3张图片，轮播展示 |
| 模块8 | 附近推荐 | 展示附近景点、美食、住宿 |
| 模块9 | 路线推荐 | 预置多条经典路线 |
| 模块10 | 用户偏好 | 首次进入选择偏好，个性化推荐 |

## ⏸️ 暂不开发

以下功能暂缓，当前阶段优先验证产品体验：

| 功能 | 原因 |
|------|------|
| 用户登录 | 优先本地存储方案 |
| Redis | 当前规模不需要 |
| PostgreSQL | SQLite 足够 |
| 向量数据库 | 当前不需要语义搜索 |
| Agent 框架 | 保持简单架构 |
| 支付系统 | 不涉及交易 |
| 酒店预订 | 超出当前范围 |
| 门票预订 | 超出当前范围 |
| 多城市扩展 | 先验证单城市模型 |
